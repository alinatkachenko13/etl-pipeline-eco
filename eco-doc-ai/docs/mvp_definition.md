# MVP проекта

## Цель MVP
Создать минимально рабочий прототип системы, которая умеет принимать документ, извлекать из него текст, находить нужные фрагменты и отвечать на вопросы по содержимому.

## Что входит в MVP
- загрузка одного документа
- поддержка базовых форматов: PDF, DOCX, TXT, JPG, PNG
- извлечение текста из текстовых документов
- OCR для изображений и сканов
- базовая очистка текста
- разбиение текста на чанки
- поиск релевантных фрагментов по запросу
- краткое саммари документа
- ответы на вопросы по документу

## Что не входит в MVP
- полноценное дообучение модели
- сложная генерация полной документации
- обработка больших корпоративных архивов в production-режиме
- сложный пользовательский интерфейс
- многопользовательская система
- глубокая экспертная валидация на большом наборе кейсов

## Пользовательский сценарий MVP
1. пользователь загружает документ
2. система определяет формат файла
3. система извлекает текст или выполняет OCR
4. текст очищается и разбивается на чанки
5. пользователь задаёт вопрос
6. система находит релевантные фрагменты
7. LLM формирует ответ на основе найденного контекста
8. пользователь получает ответ и краткое саммари

## Критерии готовности MVP
MVP считается готовым, если:
- система обрабатывает хотя бы несколько типов документов
- текст успешно извлекается хотя бы на тестовом наборе файлов
- по документу можно получить саммари
- по документу можно задать вопрос и получить ответ
- retrieval работает на базовом уровне
- результат можно показать как прототип

## Ограничения MVP
- качество ответов зависит от качества извлечённого текста
- OCR может ошибаться на плохих изображениях
- ответы могут быть ограничены рамками одного документа или малого корпуса
- точность извлечения требований на первом этапе будет базовой
