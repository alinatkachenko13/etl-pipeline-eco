# Архитектура (черновик)

## Назначение
Описание высокоуровневой логики системы Eco Doc AI в соответствии с [MVP](mvp_definition.md).

## Логические компоненты
- **Приём файлов** — загрузка документа, определение формата.
- **Извлечение текста** — парсинг PDF/DOCX/TXT; при необходимости **OCR** для изображений и сканов.
- **Обработка текста** — очистка, нормализация, **разбиение на чанки**.
- **Индексация и поиск** — хранение чанков, **retrieval** по запросу пользователя.
- **Генерация ответов** — вызов **LLM** с отобранным контекстом для **question-answering** и **саммари**.

## Поток данных
Файл → извлечение/OCR → чанки → индекс → запрос пользователя → релевантные фрагменты → LLM → ответ и/или саммари.

## Хранение
Сырые и обработанные данные — по политике проекта (например, каталоги `data/raw`, `data/processed`); артефакты экспериментов — в `results/`. Детализация уточняется по мере реализации.
